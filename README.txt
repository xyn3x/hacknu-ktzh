Digital Twin Locomotive Dashboard
--------------------------------------------------------------------------------------------------------------
Description

Цифровой двойник локомотива — это full-stack веб-приложение для визуализации телеметрии в реальном времени и оценки состояния поезда через интегральный индекс здоровья.
Проект решает проблему перегрузки данными: вместо сотен параметров пользователь получает понятную и приоритизированную картину состояния системы.
--------------------------------------------------------------------------------------------------------------
Features

Визуализация телеметрии (скорость, топливо, давление, температура, электрические параметры)
Индекс здоровья (0–100) с цветовой индикацией:
  A. Норма
  B. Внимание
  C. Критично
Система алертов и рекомендаций
Живые графики и тренды (auto-scale и zoom)
Карта маршрута с текущим положением
Просмотр истории (replay) последних 5–15 минут
Экспорт подробных отчетов (PDF / CSV)
Режим повышенной нагрузки (highload)
--------------------------------------------------------------------------------------------------------------
Architecture
Система построена на микросервисной архитектуре, что обеспечивает масштабируемость и отказоустойчивость.

Frontend → API Gateway → Backend Services → Database
                     ↘ WebSocket (real-time)
Компоненты:
- Frontend — визуализация и пользовательский интерфейс
- API Gateway — единая точка входа
- Generator - генерация данных телеметрии
- Ingestion Service — прием телеметрии
- Processing Service — обработка данных и расчет индекса
- Message Broker (Kafka / Redpanda) — потоковая передача данных
- Database (PostgreSQL / TimescaleDB) — хранение истории
--------------------------------------------------------------------------------------------------------------
Tech Stack

Система построена на микросервисной архитектуре, что обеспечивает независимое масштабирование компонентов и высокую отказоустойчивость.
- Backend: Node.js (Express/NestJS) и Go (Fiber/Gin)
- Real-time & Messaging: Kafka / Redpanda, WebSocket
- Databases: TimescaleDB / PostgreSQL
- Frontend: TypeScript / JavaScript (React/Vue/Svelte)
- Infrastructure: Docker, Docker Compose, Git
--------------------------------------------------------------------------------------------------------------
Data Pipeline

Прототип реализует полный цикл обработки потоковой телеметрии:
Прием:
- Потоковые данные поступают через WebSocket с частотой от 1 Гц и выше
Очистка:
- Буферизация, дедубликация и сглаживание шумов (EMA/медиана)
Валидация:
- Проверка входящих пакетов перед обработкой
Отказоустойчивость:
- Автоматический reconnect, стратегия backoff и индикация «Нет связи»
--------------------------------------------------------------------------------------------------------------
Health Index

Индекс здоровья — это интегральный показатель состояния локомотива:
Диапазон: 0–100
Категории:
A. Норма
B. Внимание
C. Критично
Формируется на основе:
A. скорости
B. давления
C. температуры
D. топлива
E. электрических параметров
F. наличия ошибок и алертов
Объяснимость: вклад ключевых параметров в итоговый индекс

UI Usage Guide

A. Workspace Setup
- Переключение светлой и темной темы
- Выбор языка
- Перемещение виджетов (drag-and-drop)
- Сброс расположения интерфейса
B. Monitoring and Analytics
- Индекс здоровья как основной индикатор состояния
- Живые графики с автоматическим масштабированием и zoom
- Карта маршрута и текущего положения
- Панель алертов в реальном времени
C. Testing and Reporting
- Режим повышенной нагрузки (симуляция x10 нагрузки)
- Экспорт отчетов в PDF и CSV

Getting Started

A. Docker
  git clone <repo>
  cd project
  docker-compose up --build

B. Без Docker
  # Backend
  cd backend
  go run main.go
  
  # Frontend
  cd frontend
  npm install
  npm run dev
